# StreamId Attribute

The `[StreamId]` attribute marks a property or parameter as the stream identifier for event sourcing. The stream ID identifies which stream (aggregate) a message belongs to, enabling the source generator to create zero-reflection extraction methods.

## Namespace

```csharp
using Whizbang.Core;
```

## Syntax

```csharp
// On a record parameter
public record OrderCreated([property: StreamId] Guid OrderId, string ProductName) : IEvent;

// On a property
public record CreateOrder : ICommand {
  [StreamId]
  public Guid OrderId { get; init; }
  public string ProductName { get; init; } = string.Empty;
}
```

## Applies To

- Properties on event types (implementing `IEvent`)
- Properties on command types (implementing `ICommand`)
- Record parameters (using `[property: StreamId]` target)

## Purpose

The `[StreamId]` attribute serves two critical purposes:

1. **Stream Association**: Identifies which aggregate/stream a message belongs to
2. **Compile-Time Extraction**: Source generator creates zero-reflection `ExtractStreamId()` methods for AOT compatibility

## Requirements

- **Property type**: Must be `Guid`, `Guid?`, or a WhizbangId type
- **One per type**: Only one property per message type should have this attribute
- **Inherited**: Attribute is inherited by derived message types
- **Accessible**: Property must have a `get` accessor

## How It Works

### 1. Compile-Time Discovery

The source generator finds properties marked with `[StreamId]` and generates extraction methods:

```csharp{title="Generated Extraction" description="Generated zero-reflection stream ID extraction" category="Reference" difficulty="INTERMEDIATE" tags=["StreamId", "Source-Generators"]}
// Generated by StreamIdGenerator
public static partial class StreamIdExtractor {
  public static Guid? ExtractStreamId(OrderCreated @event) {
    return @event.OrderId;  // Direct property access - no reflection
  }

  public static Guid? ExtractStreamId(UpdateOrder command) {
    return command.OrderId;
  }
}
```

### 2. Runtime Resolution

The `StreamIdResolver` uses generated extractors to resolve stream IDs:

```csharp{title="StreamIdResolver Usage" description="Resolve stream ID from any message" category="Reference" difficulty="INTERMEDIATE" tags=["StreamId", "Runtime"]}
// In your code
var resolver = serviceProvider.GetRequiredService<IStreamIdResolver>();
var streamId = resolver.GetStreamId(orderCreatedEvent);  // Returns OrderId value
```

### 3. Event Store Integration

Stream IDs enable grouping events by aggregate:

```csharp{title="Event Store Usage" description="How stream IDs enable event grouping" category="Reference" difficulty="INTERMEDIATE" tags=["StreamId", "Event-Store"]}
// Events are stored per-stream
var events = await eventStore.GetEventsForStreamAsync(orderId);

// Append to specific stream
await eventStore.AppendAsync(streamId: orderId, events: [orderShipped]);
```

## Multiple Events for Same Aggregate

All events for the same aggregate should use consistent property names:

```csharp{title="Consistent StreamId" description="Use same property name across aggregate events" category="Usage" difficulty="BEGINNER" tags=["StreamId", "Events", "Aggregates"]}
public record OrderCreated : IEvent {
  [StreamId]
  public Guid OrderId { get; init; }
  public DateTime CreatedAt { get; init; }
}

public record OrderShipped : IEvent {
  [StreamId]
  public Guid OrderId { get; init; }  // Same property name
  public string TrackingNumber { get; init; } = string.Empty;
}

public record OrderCancelled : IEvent {
  [StreamId]
  public Guid OrderId { get; init; }  // Same property name
  public string Reason { get; init; } = string.Empty;
}
```

## Diagnostics

The source generator validates `[StreamId]` usage:

### WHIZ040: Multiple StreamId Properties

**Error**: Message has multiple properties with `[StreamId]`

```csharp{title="Multiple StreamId Error" description="Error when multiple StreamId attributes are used" category="Diagnostics" difficulty="BEGINNER" tags=["StreamId", "Diagnostics", "Errors"]}
// Causes WHIZ040
public record InvalidEvent : IEvent {
  [StreamId]
  public Guid OrderId { get; init; }

  [StreamId]  // Error: Only one [StreamId] allowed!
  public Guid CustomerId { get; init; }
}

// Fixed: Choose the primary aggregate
public record ValidEvent : IEvent {
  [StreamId]
  public Guid OrderId { get; init; }  // Primary aggregate

  public Guid CustomerId { get; init; }  // Related entity, not stream ID
}
```

### WHIZ041: Invalid StreamId Type

**Error**: `[StreamId]` property is not a supported type

```csharp{title="Invalid Type Error" description="Error when StreamId type is not supported" category="Diagnostics" difficulty="BEGINNER" tags=["StreamId", "Diagnostics", "Errors"]}
// Causes WHIZ041
public record InvalidEvent : IEvent {
  [StreamId]
  public string OrderId { get; init; } = string.Empty;  // Error: string not supported
}

// Fixed: Use Guid
public record ValidEvent : IEvent {
  [StreamId]
  public Guid OrderId { get; init; }
}
```

## Differences from StreamKey

| Attribute | Purpose | Used By |
|-----------|---------|---------|
| `[StreamId]` | Identifies stream for event store | Events, Commands |
| `[StreamKey]` | Groups events for perspective ordering | Events, Models |

Both attributes often appear on the same property, as the stream ID for storage and the stream key for perspectives are typically the same:

```csharp{title="StreamId and StreamKey Together" description="Combining StreamId and StreamKey attributes" category="Usage" difficulty="INTERMEDIATE" tags=["StreamId", "StreamKey", "Combined"]}
public record OrderCreated : IEvent {
  [StreamId]   // For event store stream identification
  [StreamKey]  // For perspective event ordering
  public Guid OrderId { get; init; }
}
```

## Best Practices

### Use on Aggregate Root ID

```csharp{title="Aggregate Root ID" description="StreamId should be the aggregate root identifier" category="Best-Practices" difficulty="BEGINNER" tags=["StreamId", "Best-Practices"]}
// OrderId is the aggregate root - use StreamId here
public record OrderEvent : IEvent {
  [StreamId]
  public Guid OrderId { get; init; }

  public Guid CustomerId { get; init; }  // Related entity, not stream ID
}
```

### Consistent Property Names

```csharp{title="Consistent Names" description="Use same property name across aggregate events" category="Best-Practices" difficulty="BEGINNER" tags=["StreamId", "Best-Practices", "Naming"]}
// All order events use "OrderId"
public record OrderCreated : IEvent {
  [StreamId] public Guid OrderId { get; init; }
}

public record OrderUpdated : IEvent {
  [StreamId] public Guid OrderId { get; init; }  // Same name
}
```

### UUID7 for Time-Ordering

```csharp{title="UUID7 Recommendation" description="Use UUID7 for time-ordered stream IDs" category="Best-Practices" difficulty="INTERMEDIATE" tags=["StreamId", "Best-Practices", "UUID7"]}
// UUID7 provides time-ordering which is useful for:
// - Sequential ID generation without coordination
// - Natural ordering by creation time
// - Efficient indexing in databases

[StreamId]
public Guid OrderId { get; init; } = Guid.CreateVersion7();
```

## Zero Reflection and AOT

The `[StreamId]` attribute enables zero-reflection stream ID extraction:

**Without [StreamId] (reflection)**:
```csharp
// Runtime reflection - not AOT-compatible
var streamId = @event.GetType()
    .GetProperty("OrderId")
    ?.GetValue(@event) as Guid?;
```

**With [StreamId] (generated)**:
```csharp
// Compile-time code generation - AOT-compatible
public static Guid? ExtractStreamId(OrderCreated @event) {
  return @event.OrderId;  // Direct property access
}
```

Benefits:
- Works with Native AOT
- Zero runtime overhead
- Type-safe at compile-time
- No reflection allocations
