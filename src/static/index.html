What is Whizbang?

Whizbang is a unified event-sourced data and messaging runtime for .NET that collapses MartenDB, Wolverine, MassTransit, and MediatR into one cohesive platform with a single mental model.

Key Features

  • Zero Reflection - Everything via source generators for AOT compatibility
  • Poly-Store Adapters - PostgreSQL, SQL Server, MySQL, Cosmos DB, SQLite support
  • Unified Messaging - Commands, queries, events, and sagas with single API
  • Perspectives (Projections) - Real-time read models with typed lenses
  • Built-in Observability - OpenTelemetry traces and metrics by default
  • Transport Agnostic - Azure Service Bus, RabbitMQ, or in-memory

Quick Start Example

// Define a message
public record OrderPlaced(Guid OrderId, string CustomerId, decimal Total);

// Handle with a Receptor
public class OrderPlacedReceptor : IReceptor<OrderPlaced>
{
    public async Task HandleAsync(OrderPlaced message, CancellationToken ct)
    {
        // Process the order
        Console.WriteLine($"Order {message.OrderId} placed for {message.Total:C}");
    }
}

// Configure in Program.cs
builder.Services.AddWhizbang(wb => {
    wb.UsePostgres(connectionString);
    wb.UseAzureServiceBus(serviceBusConnection);
    wb.DiscoverReceptorsFromAssembly(typeof(Program).Assembly);
});

Core Concepts

  • Receptors - Message handlers (sync and async)
  • Dispatcher - Routes messages to appropriate receptors
  • Perspectives - Projections/read models with typed access via lenses
  • Event Store - Append-only event storage with stream support
  • Transports - Azure Service Bus, RabbitMQ, or in-memory messaging
  • Source Generators - Compile-time code generation for zero reflection
